## 1. 3職能に共通する人材要件フレーム

すべての職種に共通して「3層構造」で整理されている：

```
【第1層】マインドセット（根）
　　　↓
【第2層】知識・スキル（幹）
　　　↓
【第3層】経験（実）
```

各層を知識（K）・スキル（S）・経験（E）・マインドセット（M）の4軸 ＋ 職種特化要件（P/D）で整理。

## 3. プロデューサー ToBeモデル（v3）

### ToBe像
> **「日本にしかない物語を理解し、アーティストを守り、世界と繋げることができるプロデューサー」**

### 業界課題（育成が必要な理由）
- 日本にプロデューサーを学べる大学教育がほぼない（韓国・ロンドン・NYと比較して顕著）
- 企画書・助成金申請・契約など制作実務スキルが芸術教育から抜け落ちている
- 「良い作品があれば売れる」という素朴な信仰が根強く、戦略的思考が育ちにくい
- 才能を育てても発表・デビューまでのpathway（道筋）が整備されていない

### 最優先要件（◎ 11項目）

| ID | 軸 | 要件 |
|---|---|---|
| M-1 | マインドセット | コラボレーションを喜びとして捉える姿勢 |
| M-2 | マインドセット | 自分の文化への誇りと相対化能力 |
| M-3 | マインドセット | 届けたい・売りたいという欲望 |
| K-1 | 知識 | 演劇の構造・ドラマトゥルギー基礎 |
| K-2 | 知識 | 日本の文化・伝統芸能への理解 |
| K-3 | 知識 | 制作実務・プロデュース基礎知識 |
| S-1 | スキル | コラボレーション技術 |
| S-2 | スキル | 英語コミュニケーション・ピッチ力 |
| S-3 | スキル | 作品・プロジェクトの価値言語化 |
| E-1 | 経験 | 現場体験（観客・作り手） |
| E-2 | 経験 | 協働創作の実践経験 |

### プロデューサー特化要件（P機能）

| ID | 要件 |
|---|---|
| P-1 | 戦略的思考（市場・タイミング・フォーマットを考える） |
| P-2 | アーティストの守り手（才能を演劇界に「残す」使命感） |
| P-3 | ネットワーク構築力（クリエイター・劇場・資金者・翻訳者をつなぐハブ力） |
| P-4 | 劇作家・演出家マッチング能力 |
| P-5 | 継続する力（長期的関係・信用の蓄積） |

### リサーチ根拠5つの示唆

1. 育成とpathwayはセットで設計せよ
2. レジデンシー的仕組みが有効（劇場のデスク1つが作家のキャリアを変える）
3. 「売りたい人」と「作りたい人」のマッチングが鍵
4. 小さく始めて継続する（ドゥサン20年、Out of the Wings 10年）
5. 共通言語の整備が協働の土台

## 4. 演出家 ToBeモデル（v1）

### ToBe像
> **「作品の探求を導き、多様なアーティストと協働しながら、日本の物語を世界に届けられる演出家」**

### 業界課題（育成が必要な理由）
- 日本に演出家を専門的に育成する大学教育がほぼない
- コラボレーションではなくトップダウンの演出スタイルが根強く残っている
- 海外の俳優・スタッフと働くための共通言語・メソッドの知識が不足
- 演出家のキャリアパスが不明瞭でメンターとの出会いが偶然に委ねられている
- 文化的翻訳・コンテクスト化の技術が育っていない

### 最優先要件（◎ 13〜14項目）

| ID | 軸 | 要件 |
|---|---|---|
| M-1 | マインドセット | 探求を導く姿勢（答えを押しつけない） |
| M-2 | マインドセット | カンパニー参加者へのリスペクト |
| M-3 | マインドセット | 愛——作品・演劇・人への |
| K-1 | 知識 | 戯曲・テキストを読み込む力 |
| K-2 | 知識 | 演技メソッド・共通言語の理解（ラバン等） |
| K-3 | 知識 | 演劇史・世界の演出家への理解 |
| K-4 | 知識 | 日本の文化・伝統芸能・精神的価値観 |
| S-1 | スキル | 聞く力（俳優・スタッフの声を受け取る） |
| S-2 | スキル | コラボレーション技術（全職種と対等に対話する） |
| S-3 | スキル | 作品のビジョンを言語化・共有する力 |
| S-4 | スキル | 稽古場のリーダーシップ・空気づくり |
| E-1 | 経験 | 優秀な演出家のアシスタント経験 |
| E-2 | 経験 | 実際に演出する経験（規模問わず） |
| E-3 | 経験 | 多様な職種・分野との協働創作体験 |
| E-4 | 経験 | 現場体験（劇場で働く・見る） |

### 演出家特化要件（D機能 / キャリア形成支援）

| ID | 要件 |
|---|---|
| D-1 | メンターとの出会いを設計する（偶然に委ねない） |
| D-2 | 小さくても演出できる場の確保 |
| D-3 | 劇作家・プロデューサーとのマッチング機会 |
| D-4 | アシスタント・インターンシップの制度化 |
| D-5 | 海外演出の実践機会 |

### リサーチ根拠5つの示唆

1. 演出家はメンターと実践でしか育たない（大学教育には限界）
2. キャリアを急がせるな（ブロードウェイデビューは40代で遅くない）
3. コラボレーションは技術として教えられる
4. 他言語・異文化での演出が感度を上げる
5. 日本固有の感性は武器だが、普遍的な人間ドラマに昇華する技術が必要

## 5. 劇作家 ToBeモデル（v1）

### ToBe像
> **「普遍的な人間の物語を、日本固有の感性で書き、世界に届けられる劇作家」**

書く技術だけでなく、演劇全体を知り、他者と協働し、自分の作品の価値を言語化して届けることができる人材。

### 業界課題（育成が必要な理由）
- 劇作を専門的に学べる大学がほぼ2〜3校しかない（韓国・イギリスと比較して顕著）
- 書き方は教わるが、演劇全体のプロセス・現場を知らないまま書き始めることが多い
- 劇作家がデビュー後も活動を継続できるpathway（道筋）が整備されていない
- 助成金申請・企画書・契約など制作実務を学ぶ機会がない
- 劇作家と演出家・プロデューサーとのマッチングの仕組みがない

### 最優先要件（◎ 12〜14項目）

| ID | 軸 | 要件 |
|---|---|---|
| M-1 | マインドセット | 書き続ける意志・作家としてのアイデンティティ |
| M-2 | マインドセット | 自分の文化・社会への深い関心 |
| M-3 | マインドセット | 他者と協働することへの開放性 |
| K-1 | 知識 | 演劇の構造・戯曲の論理（ドラマトゥルギー） |
| K-2 | 知識 | 古典・同時代の戯曲を読む習慣 |
| K-3 | 知識 | 演劇全体のプロセス理解（書く以外の現場） |
| K-4 | 知識 | 日本文化・伝統芸能・社会史への理解 |
| K-5 | 知識 | 助成金・企画書・契約の実務知識 |
| S-1 | スキル | 書く技術（セリフ・構造・シーン） |
| S-2 | スキル | 自作を語る・言語化する力 |
| S-3 | スキル | 他者のフィードバックを受け取る力 |
| E-1 | 経験 | 演劇の現場を体験してから書く |
| E-2 | 経験 | 書いた作品が俳優に声に出される経験 |
| E-3 | 経験 | 定期的に書き続ける習慣・反復練習 |

### 劇作家特化要件（継続・デビュー支援）

| ID | 要件 |
|---|---|
| D-1 | pathway（上演までの道筋）の整備 |
| D-2 | ドラマターグ・メンターとの関係（守る仕組み） |
| D-3 | 演出家・劇場とのマッチング機会（競争より出会いの場） |
| D-4 | 経済的基盤との両立（劇作だけで生きなくても書き続けられる選択肢） |
| D-5 | 戯曲の出版・アーカイブ（英訳・リーディングの積み重ねが次の上演へ） |

### リサーチ根拠5つの示唆

1. 書く前に現場を見せよ（ミン・セロム・前川・キム・マディン）
2. 遠回りは武器になる（ユン・ジュホ：社会人経験後に劇作、ファン・スア：詩人→劇作家）
3. 守る仕組みと育てる仕組みはセット（ARKOの設計思想：才能ある劇作家が業界を離れないよう守る）
4. 競争より出会いの場を（演出家・俳優・劇場との平和的マッチングの場が必要）
5. 戯曲を残す・届ける仕組みが連鎖を生む（英訳・出版・リーディング→次の上演・海外展開）

## 6. 3職種横断の選考基準への変換（暫定）

人材要件を選考に落とし込む際の観点：

| 選考段階 | プロデューサー観点 | 演出家観点 | 共通観点 |
|---|---|---|---|
| 書類 | 届けたい・売りたい欲望の記述 | 演出への強い動機・ビジョン | 自分の文化への視点 |
| テスト | 作品価値言語化・企画書的思考 | 戯曲読解・分析 | ドラマトゥルギー基礎 |
| 面接 | コラボレーション姿勢・英語意欲 | 聞く姿勢・チームへの関心 | 長期的視野 |

→ 選考の詳細設計は `selection` エージェントに引き継ぐ

## 7. このエージェントの優先タスク

1. **劇作家ToBeモデルの作成**（最優先：3職種が揃わないとサマースクール設計が完結しない）
2. 3職種横断の共通要件を「一般教養講座」の内容に変換（→ `general-education` へ）
3. 各職種の最優先要件をカリキュラム学習目標に変換（→ 各職種プログラムエージェントへ）
4. 選考基準の言語化支援（→ `selection` へ）
5. To-be/As-Is差分の整理（リサーチエージェントと協働）
